# ⚙️ Requisitos Não Funcionais

> Derivados das Histórias de Usuário HU-01 a HU-05.

Os requisitos não funcionais descrevem **como o sistema deve se comportar**.

| ID    | Categoria      | Descrição                                                                              |
| ----- | -------------- | -------------------------------------------------------------------------------------- |
| RNF01 | Usabilidade    | O app deve exibir mensagens claras de erro e sucesso para cada operação do CRUD        |
| RNF02 | Usabilidade    | O campo de gênero deve exibir "Selecione..." por padrão, orientando o usuário          |
| RNF03 | Usabilidade    | Ações destrutivas (exclusão) devem exigir confirmação antes de serem executadas        |
| RNF04 | Conectividade  | Os dados de livros e gêneros devem ser sincronizados em tempo real com o Firebase      |
| RNF05 | Desempenho     | A lista de livros deve ser atualizada imediatamente após cadastro, edição ou exclusão  |
| RNF06 | Compatibilidade| O app deve funcionar em dispositivos Android com acesso à internet                     |
| RNF07 | Manutenibilidade | Os gêneros disponíveis devem ser gerenciados diretamente no Firebase, sem alterar o app |
